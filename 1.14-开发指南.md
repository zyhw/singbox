# sing-box 1.14 分支开发指南

本文档为 `1.14` 分支的开发说明，用于后续将 sing-box 从 1.12.x 迁移至 1.14.x。

---

## 分支结构

| 分支 | 用途 | 状态 |
|------|------|------|
| `main` | sing-box 1.12.x 稳定版 | ✅ 生产环境使用中 |
| `1.14` | sing-box 1.14.x 开发适配 | 🔧 开发中，待稳定后合并至 main |

> 切换命令：`git checkout main` / `git checkout 1.14`
> 两个分支的文件互不影响。

---

## 1.14 需要适配的变更清单

根据 [官方迁移文档](https://sing-box.sagernet.org/migration/#1140)，以下是从 1.12 升级到 1.14 需要处理的内容：

### 1. 脚本层面 (`sing.sh` / `upgrade.sh`)

- [ ] 将版本号从 `1.12.*` 改为 `1.14.*`
- [ ] GitHub API 获取 tag 的正则从 `v1.12.` 改为 `v1.14.`
- [ ] README 同步更新版本说明

### 2. 配置文件层面 (`config.json`)

#### 2.1 必须迁移：`route.default_domain_resolver`（1.12 弃用 → 1.14 移除）

当前 1.12 配置中的 DNS 部分使用了旧格式，在 1.14 中将被彻底移除。需要增加 `route.default_domain_resolver` 字段。

**当前 1.12 配置（会产生 WARN 告警）：**
```json
{
  "dns": {
    "servers": [
      { "tag": "local", "address": "local" },
      { "tag": "proxy", "address": "1.1.1.1" }
    ],
    "rules": [
      { "rule_set": "geosite-cn", "server": "local" }
    ],
    "final": "proxy",
    "strategy": "ipv4_only"
  }
}
```

**迁移后 1.14 配置：**
```json
{
  "dns": {
    "servers": [
      { "type": "local", "tag": "local" },
      { "type": "udp", "tag": "proxy", "server": "1.1.1.1" }
    ],
    "rules": [
      { "rule_set": "geosite-cn", "server": "local" }
    ],
    "final": "proxy",
    "strategy": "ipv4_only"
  },
  "route": {
    "default_domain_resolver": {
      "server": "local",
      "strategy": "ipv4_only"
    }
  }
}
```

#### 2.2 可选注意：ACME 内联配置迁移至 `certificate_provider`

如果使用了 TLS ACME（当前脚本使用 Reality，**不涉及此项**），需将 `tls.acme` 迁移为 `tls.certificate_provider`。

**当前脚本不使用 ACME，因此无需修改此项。**

## 开发流程

```bash
# 1. 确保在 1.14 分支上
git checkout 1.14

# 2. 修改脚本和配置...

# 3. 提交并推送
git add . && git commit -m "feat: adapt for sing-box 1.14" && git push

# 4. 稳定后合并回 main
git checkout main
git merge 1.14
git push
```
